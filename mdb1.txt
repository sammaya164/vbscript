<p>データベースへ接続するための事前準備として、利用可能なOLEDBプロバイダーを確認する。</p>
<h2 id="oledbプロバイダーを確認する">OLEDBプロバイダーを確認する</h2>
<p>PowerShellを起動して以下のコマンドを実行する。</p>
<pre><code class="hljs language-shell">(New-Object data.oledb.oledbenumerator).getElements() | select SOURCES_NAME, SOURCES_DESCRIPTION
</code></pre>
<p>64bit版のOLEDBが表示される。必要な行だけ示す。</p>
<pre><code class="hljs language-shell">SOURCES_NAME               SOURCES_DESCRIPTION
------------               -------------------
Microsoft.ACE.OLEDB.12.0   Microsoft Office 12.0 Access Database Engine OLE DB Provider
Microsoft.ACE.OLEDB.16.0   Microsoft Office 16.0 Access Database Engine OLE DB Provider
</code></pre>
<p>つぎに32bit版PowerShellを起動して同じコマンドを実行する。32bit版PowerShellは<code>C:\Windows\SysWOW64\WindowsPowerShell\v1.0\powershell.exe</code>にある。</p>
<p>32bit版のOLEDBが表示される. 必要な行だけ示す。</p>
<pre><code class="hljs language-shell">SOURCES_NAME               SOURCES_DESCRIPTION
------------               -------------------
Microsoft.Jet.OLEDB.4.0    Microsoft Jet 4.0 OLE DB Provider
</code></pre>
<p>以上から自分の環境では</p>
<ul>
<li>64bit版の<code>Microsoft.ACE.OLEDB.12.0</code></li>
<li>64bit版の<code>Microsoft.ACE.OLEDB.16.0</code></li>
<li>32bit版の<code>Microsoft.Jet.OLEDB.4.0</code></li>
</ul>
<p>が利用できることが分かる。</p>
<p>VBScriptを動かしているWSHにも32bit版と64bit版がある。</p>
<p>64bit版WindowsではスクリプトをWクリックしたとき、とくに設定を変えてなければ64bit版が起動する。<br>
作業フォルダに<code>test.vbs</code>があるとして、ターミナルから以下のように実行したときも64bit版が起動する。</p>
<pre><code class="hljs language-vbs">C:\Windows\System32\wscript.exe .\test.vbs
</code></pre>
<p>以下のように実行したときは32bit版が起動する。</p>
<pre><code class="hljs language-vbs">C:\Windows\SysWOW64\wscript.exe .\test.vbs
</code></pre>
<p>32bit版を起動するのにコマンドを打つのが面倒な場合は、下記のスクリプトにドラッグ＆ドロップすると良い。</p>
<pre><code class="hljs language-vbs"><span class="hljs-comment">'このスクリプトファイルにドラッグ&#x26;ドロップしたスクリプトファイルを32bit版のWSHで実行する</span>
